### 重要な発見

1. **Gemini API の非決定性**: 同じクエリでも、翻訳結果が毎回異なるため、検索結果も変動する
2. **キーワード数の最適範囲**: 10-13 語程度が最も効果的である可能性が高い
3. **専門用語の重要性**: "intraflagellar transport (IFT)" などの専門用語を含めることで、検索精度が向上する
4. **検索エンジンの違い**: Semantic Scholar と PubMed で結果が大きく異なる場合がある

### 問題点

1. **非決定性**: Gemini API の出力が毎回異なるため、同じクエリでも結果が変わる
2. **キーワードの過剰拡張**: 場合によってはキーワードが多すぎて、結果が減る
3. **検索エンジン間の差異**: Semantic Scholar と PubMed で結果の質が異なる

### 改善案

1. **キーワード数の最適化**: 10-12 語程度を目標とする
2. **専門用語の優先**: 必須の専門用語（例: IFT, ciliogenesis）を優先的に含める
3. **英語クエリの処理**: 英語の文章や論文タイトルの場合、元のキーワードを核として、最小限の拡張のみ行う
4. **検索エンジン別の最適化**: Semantic Scholar と PubMed で異なるキーワード戦略を検討
5. **キャッシュの活用**: 同じクエリの翻訳結果をキャッシュして、非決定性を軽減

### 推奨方針

**日本語クエリの場合:**

- パターン 4 の意図理解ベースのプロンプトを使用
- キーワード数を 10-12 語程度に制限
- 専門用語を優先的に含める

**英語クエリの場合:**

- 元のクエリのキーワードを核として使用
- 最小限の関連語（2-3 語）を追加するに留める
- 過剰な拡張は避ける

**論文タイトルの場合:**

- タイトルから主要なキーワードを抽出
- 種名や特定の遺伝子名は必ず含める
- 拡張は最小限に

### 総合分析

#### 引用符の効果が大きい場合

1. **複合語やタンパク質名**: 「adenylate kinase」「motor protein」「ATP synthase」など

   - 分割されると意味が変わる
   - 引用符で囲むことで、正確なフレーズ検索が可能

2. **専門用語の組み合わせ**: 「intraflagellar transport」など
   - 既に専門用語として認識されている場合でも、引用符で囲むことでより正確な検索が可能

#### 引用符の効果が小さい場合

1. **単一の単語**: 「KIF17」などの遺伝子名

   - 単一の単語なので、引用符の効果はない

2. **一般的な専門用語**: 「intraflagellar transport」など
   - 検索エンジンが既に認識している場合、引用符の有無による差は小さい

#### 推奨事項

1. **複合語やタンパク質名は引用符で囲む**: 「adenylate kinase」「motor protein」「ATP synthase」など
2. **専門用語の組み合わせも引用符で囲む**: 「intraflagellar transport」「ciliary function」など
3. **単一の単語は引用符不要**: 「KIF17」「kinesin」など
4. **プロンプトでフレーズ保持を指示**: Gemini API のプロンプトで、複合語を引用符で囲むよう指示する
